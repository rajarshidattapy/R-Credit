# R-Credit - Buidl on CTC Ideathon, India

R-Credit solves the fundamental problem of credit in decentralized systems: users can create unlimited wallets to escape bad credit history. It creates a non-escapable, privacy-preserving credit identity using zero-knowledge proofs.

## ğŸ”‘ Key Features

### 1. Non-Escapable Identity

- One identity per human using PAN card + device fingerprint
- ZK proof commits identity to Merkle tree on-chain
- Privacy-preserving: no personal data stored

### 2. Multi-Wallet Support

- Users can use unlimited wallet addresses
- All credit activity resolves to single identity
- Cannot reset credit by creating new wallets

### 3. Protected Deposit Wallet

- Borrowed funds go to protocol-controlled wallet
- Prevents "borrow and run" scenarios
- Forces repayment before full access

### 4. RWA Tokenization

- Real estate, agricultural commodities, carbon credits, art
- Issuers bound to verified identities
- Default on any asset impacts credit across all assets

### 5. Credit Scoring System

- On-chain, transparent credit scores
- Based on repayment history
- Better rates for better behavior
- Defaults are permanent and visible
